> **문서 위상**: 이 문서는 `slug-mcp`의 자격판정·가점계산 룰 엔진이 참조하는
> **정책 기준(Source of Truth 후보)**이다. 실행·배포는 [../README.md](../README.md),
> 제품 배경은 [product-concept.md](product-concept.md)를 본다.
>
> **주의(반드시 읽을 것)**: 여기 적힌 수치는 정책 기획서 + 2026-07-05 시점 웹 검증을
> 거친 값이다. **실제 판정 로직에 반영하는 값의 최종 SOT는 항상 (a) 국가법령정보센터
> 원문 별표, (b) 청약홈/LH 공고 원문**이며, 이 문서가 아니다. 각 규칙의 `검증` 태그로
> 근거 수준을 표시했다. 법령 개정 시 이 문서와 `config/eligibility_rules.yaml`을 함께 갱신한다.
>
> **📋 2026-07-06 데이터 정합성 전수조사**: 본 명세와 구현 전체를 공식 SoT(법령
> 원문·청약홈·LH·공고문)와 대조한 결과가 [data-integrity-audit.md](data-integrity-audit.md)에
> 있다. 아래 §0 표의 일부 태그는 그 결과로 갱신되었다 — 오류로 판정된 값은 감사 문서의
> 항목 번호(B-1 등)를 따라가면 "무엇이 어떻게 틀렸는지"를 볼 수 있다.
>
> **✅ 2026-07-06 수정 완료**: 위 감사가 지적한 오류 대부분을 코드에 반영했다 — 규제지역
> 15곳, 분양 소득표 교체("3인 이하" 통합·가산 579,278), 특공 소득·통장·자산 게이트, 신혼
> 일반형 별표6 순위제+13점, 다자녀 통장 10년↑ 5점 단일, 무주택 만30세 기산, 월 25만원
> 소급, 세종 tier3 등. 아래 §0 표의 태그는 **감사 시점 기준**이며, 항목별 처리 결과는
> [data-integrity-audit.md](data-integrity-audit.md)의 "처리 상태"를 본다. 미구현 잔여분은
> B-13(미성년 통장 5년)·B-14(민영 신생아 특공, 2026-06-15 신설)뿐이다.

## 1. 입력 스키마 (Input JSON)

룰 엔진은 자연어를 파싱하지 않는다. MCP 클라이언트(LLM)가 아래 구조로 정규화해 넘긴다.
필드명은 구현부(`models.py`의 `ProfileDocument`)와 1:1 매핑된다. 구현에서는 모든 필드가
선택 입력이며, 대화 몇 번에 걸쳐 `update_my_profile`로 조각조각 채워진다 — 필수 항목이
비면 엔진이 "물어볼 질문 목록"을 돌려준다. 다자녀 세대구성 가점용
`is_three_generation_household`/`is_single_parent` 두 필드는 구현에서 추가되었다.

    "is_forced_matching": false,     // 점수 미달이어도 실현가능성 분석을 원하는지

## 2. 판정 파이프라인 (전체 흐름)

```
[입력] → 1단계 Hard Filter (부적격 즉시 반환)
       → 2단계 트랙 분기 (공공 / 민영, 유형별 매칭)
       → 3단계 가점·배점 연산
       → 4단계 목표지역 컷오프 대조 + (강제매칭 시) 우회로 분석
       → [출력 JSON]
```

## 3. 1단계 — 자격 차단 필터 (Hard Eligibility Filter)

하나라도 걸리면 해당 트랙의 점수 연산을 생략하고 `disqualification_reasons`에 사유를 담아 반환한다.

### Filter-01 · 무주택 요건 (세대 단위)  `검증: 🟡`
- **판정 단위**: 개인이 아니라 **무주택세대구성원**(주민등록등본상 세대 전원 무주택). 미혼이라도
  부모와 같은 등본이면 부모 주택이 세대에 귀속된다.
- **조건**: `engine.homeless_household_member(...)`가 유주택으로 판정하면 차단. 세대 주택 보유는
  `owned_house_count > 0`(미입력 시 `homeless_duration_months == 0` 프록시)로 본다. **단**
  그 주택 소유자가 본인이 아니라(`owns_home_self != true`) 만 60세 이상 직계존속
  (`home_owner_is_ascendant_60plus == true`)이면 **주택공급규칙 제53조로 무주택 간주**한다.
- **결과**: 유주택 세대면 **공공분양 전체(일반+특공)**, **민영 특별공급 전체** 진입 불가.
  판정 결과는 `eligibility_status.is_homeless_household_member`/`homeless_status_basis`로 노출한다.
- **가점 정합성**: 유주택 세대로 판정되면 `effective_homeless_months=0`으로 눌러 무주택기간
  가점이 붙지 않게 한다(입력값과 무관).
- **60세 예외 범위**: 엔진이 구현한 트랙(공공/민영 일반공급·신생아/신혼/다자녀/생애최초 특공)은
  전부 제53조 예외 적용 대상이다. 예외의 예외인 노부모부양 특공·공공임대는 미구현이라 충돌 없다.
- **주의**: **민영주택 일반공급(가점제)은 유주택자도 신청 가능**하다(비규제지역) — 배제가 아니라
  무주택기간 점수가 0~2점으로 낮아질 뿐(§3.B.①). 규제지역 유주택 세대는 가점제에서도 제외된다
  (규칙 제28조⑥).

### Filter-02 · 공공분양 자산 컷오프 (전용 60㎡ 이하 신청 시)  `검증: 🟡 원문 대조`
- **조건 A**: `total_real_estate_krw > 215,500,000` → 부동산 자산 초과
- **조건 B**: `car_value_krw > 45,420,000` → 자동차 가액 초과
- **결과**: 공공분양 60㎡ 이하 트랙 진입 불가.
- **주의**: 이 값은 **공공분양(분양)** 기준이다. 매입임대·전세임대 자산기준(고시
  제2025-560호, 자동차 4,563만원 등)과 **다른 트랙**이므로 혼용 금지. 최종값은
  청약홈 입주자모집공고 원문의 "자산보유 기준" 표로 확정한다.

### Filter-03 · 규제지역 1순위 세대주 요건  `검증: 🟡 범위 주의`
- **조건**: `target_region`이 규제지역(투기과열지구/조정대상지역)이고
  `is_head_of_household == false`(세대원)
- **결과**: 규제지역 민영 1순위 및 공공 일반공급 1순위 진입 불가.
- **주의**: 규제지역 목록을 **"강남3구·용산"으로 하드코딩하지 말 것.**
  2025-10-15 대책으로 **서울 전역(25개구) + 경기 12곳**으로 확대됐고, **2026-07-01
  화성 동탄구·용인 기흥구·구리 추가로 현행 경기 15곳**이다. 지역 지정 현황은 국토부
  고시를 주기적으로 반영하는 **동적 목록(config)**으로 관리한다.

## 4. 2단계 [BRANCH A] — 공공분양 (국민주택) 룰 엔진

선정 기준은 **저축 인정 총액**과 **납입 횟수**이며, 특별공급은 유형별 배점표를 적용한다.

### A. 일반공급 당첨선 (순차제)  `검증: 🟡 원문 대조`
- 3년 이상 무주택 세대구성원 중 **저축 총액(`total_balance_krw`)** 높은 순 선정.
- **전용 40㎡ 이하**는 저축 총액이 아닌 **납입 횟수(`payment_count`)** 높은 순 선정.
- 월 납입 인정액 상한 **25만원** (2024-11 개정, 이후 납입분 적용).
- 미성년자 시절 납입분 인정 한도 → 🔴 **미검증**(§0 참조), 원문 확인 전 반영 보류.

### B. 특별공급 유형별 매칭·배점 엔진

#### ① 신생아 특별공급 (공공)  `검증: 🟢 물량 / 🟡 소득수치`
- **자격**: `has_child_under_2 == true` (혼인 무관, 태아·입양 포함)
- **물량 배분 + 소득기준** (LH청약플러스 확인):
  | 단계 | 물량 | 소득기준(도시근로자 월평균) | 맞벌이 |
  |---|---|---|---|

- **주의**: "100% 추첨"이 **아니다.** 우선·일반(90%)은 배점표(가구소득·자녀수·거주기간·

## 5. 2단계 [BRANCH B] — 민영주택 룰 엔진

지역별 예치금 충족 가입자 중 **가점제(84점 만점)** + **추첨제**로 선정한다.

### A. 1순위 예치금 충족 필터  `검증: 🟢`
`residence_area` × `desired_size_sqm`로 필요 예치금을 조회, `total_balance_krw`가
미달이면 1순위 자격 박탈(감점 아님). 단위: 만원.

| 거주지 | 85㎡↓ | 102㎡↓ | 135㎡↓ | 모든 면적 |
|---|---|---|---|---|
| 서울·부산 | 300 | 600 | 1,000 | 1,500 |
| 기타 광역시 | 250 | 400 | 700 | 1,000 |
| 기타 시·군 | 200 | 300 | 400 | 500 |

### B. 일반공급 가점 산식 (84점 만점)

> 아래 의사코드는 그대로 룰 엔진에 이식 가능하도록 작성했다.

#### ① 무주택 기간 — 최대 32점  `검증: 🟢`
```python
def score_homeless(p):
    # 만 30세(또는 30세 이전 혼인신고일)부터 산정
    if not p.is_married and p.age < 30:
        return 0
    years = p.homeless_duration_months // 12
    if years < 1:   return 2
    if years >= 15: return 32
    return years * 2 + 2   # 1년당 2점, 기본 2점
```

#### ② 부양가족 수 — 최대 35점  `검증: 🟢`
```python
def score_dependents(p):
    # 0명=5점, 이후 1명당 +5, 6명 이상에서 35점 상한
    return min(35, p.dependents_count * 5 + 5)
```

#### ③ 청약통장 가입기간(본인+배우자 합산) — 최대 17점  `검증: 🟢`
```python
def score_subscription(a):
    # 본인
    y = a.duration_months // 12
    if a.duration_months < 6:  self_score = 1
    elif y < 1:                self_score = 2
    elif y >= 15:              self_score = 17
    else:                      self_score = y + 2
    # 배우자(가입기간 50% 인정, 최대 3점) — 시행일 2024-03-25
    sy = a.spouse_duration_months // 12
    if a.spouse_duration_months == 0: spouse_score = 0
    elif sy < 1:  spouse_score = 1
    elif sy < 2:  spouse_score = 2
    else:         spouse_score = 3
    return min(17, self_score + spouse_score)   # 합산 상한 17
```

- **동점자 처리** `검증: 🟡`: 최종 가점 동점 시 청약통장 가입기간(`duration_months`)이
  더 긴 후보 우선. (관행적 세부순위 — "신설 법안" 여부는 미검증, 순위 로직 자체만 사용)

### C. 민영 신생아 특별공급  `검증: 🔴 미검증`
- **자격**: 모집공고일 기준 2세 미만 자녀(혼인 무관)
- **소득**: 도시근로자 월평균 130% 이하(맞벌이 160% 이하) — 수치 미검증
- **물량**: 우선 50% / 일반 20% / 추첨 30% — **출처 미확보, 구현 전 청약홈 공고 확인 필수**

## 6. 4단계 — 목표지역 컷오프 대조 & 강제 매칭 엔진

> 아래 컷오프는 **참고 기준선(planning estimate)**이며 법정 기준도, 관측된 실제 컷도 아니다.
> `analyze`는 이 정적 밴드를 **내부 판정 기준선**으로만 쓰고, 출력에
> `cutoff_basis: "planning_estimate"`를 붙여 예측이 아님을 명시한다(추천 트랙 사유도 "참고
> 기준선(관측값 아님)"으로 표기). **실제 과거 실적**은 `recommend_housing`이 같은 시군구·트랙의
> 마감 공고 **경쟁률**로 제시한다(확률 추정 대신 실측). ⚠️ 이 정적 밴드를 걷어내면
> 실현가능성·가점제/추첨제 분기·대안지역·강제매칭이 모두 근거를 잃으므로 제거하지 말고
> **라벨만 격하**한다.

### A. 지역 등급별 참고 컷오프 기준선 (2026 계획 추정치, 관측값 아님)

| 등급 | 대표 지역 | 민영 가점 컷오프 | 공공 저축총액 컷오프 |
|---|---|---|---|
| S | 서울 강남3구·용산·마포·성동 등 | 69~74 | 2,200만↑ |
| A | 서울 일반, 하남·과천 등 핵심 | 59~68 | 1,700~2,100만 |
| B | 수도권 일반 경기·인천, 광역시 주요 | 40~58 | 1,200~1,600만 |
| C | 비규제 지방 중소도시 | 무관 | 800만 미만 |

### B. 강제 매칭 처리 (is_forced_matching == true)

1. **격차 분석**: `gap = target_cutoff - user_score`
2. **우회로 가이드 분기**:
   - **Case 1 — 가점 낮으나 소득/자산 요건 충족**: 생애최초·신생아 등 **추첨형 특공** 전환 권장.
   - **Case 2 — 가점 낮고 특공 자격도 없음**: 전용 **85㎡ 초과 대형** 또는 **추첨제 비율
     높은 단지**(추첨 60~100%) 공략.
   - **Case 3 — 2세 미만 자녀 있음**: **신생아 특별공급**을 목표지역 1순위 타깃으로.
   - **Case 4 — S/A급 희망하나 점수·저축 부족**: 경기 3기 신도시(사전/본청약), GTX 호재
     외곽을 대안으로 매칭.
